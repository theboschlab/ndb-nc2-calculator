Enable Fault Tolerance (N+1) i How Nutanix N+1 works

Nutanix uses a distributed storage fabric (AOS) that replicates data across nodes with RF2 (Replication Factor 2). In an N+1 cluster, one additional node is reserved as a hot standby: if any node fails, the cluster continues operating without data loss and without manual intervention.

The spare node is counted in the hardware cost but excluded from available capacity — CPU, RAM and storage utilization shown are based on the active nodes only.
A Fault Tolerant cluster can lose one node without any data loss. Requires one additional node reserved for recovery — counted in cost but not in available capacity.
